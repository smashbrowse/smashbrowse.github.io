What is Smash Browse?

Smash Browse saves you from daily online chores by automating your Chrome web browser.

Other Web Automation Tools blindly record your web actions, often identifying html elements explicitly by using XPath identifiers, CSS selectors, or element ids. This technique is unreliable as it breaks the recorded automation when the web page changes even in the most simplest of ways, such as when the element ids change.

Smash Browse uses a new technique to capture more of the symantic meaning and intent behind your actions. Smash Browse identifies elements using only symantic information about the element, such as its text, and by using constraints to locate elements, such as its proximity to other elements. Because of this, Smash Browse automations are not tightly coupled to the implementation (xpath, css, ids) of the web page.

Nodes

A node represents a task that Smash Browse will perform for you. A node will perform some type of computation on its inputs in order to produce some outputs. For example a node could add two numbers together. In this case the inputs are the two numbers and the output is the sum. (1+1=2)

The name of the node appears directly on the node. In this case we see that the name is "data".

A node shows its inputs as boxes and its outputs as upside down triangles. The data types that flow into an input or flow out of an output are modelled after javascript data types. The complete set of data types are: numbers, strings, booleans, objects and arrays.


Editing Inputs

When you edit a node, you are editing the default values of the inputs on that node. The default value plays an important role in determining the final value of the input.

When editing a node, you can also expose or hide any of the inputs. When an input is exposed, a link can direct data from an upstream node into the input.

The current node being edited will be marked on the right with the letter "E".

These are the rules that determine how the default value affects the final value of the input.

  • If the input is not linked to any other node, then the default value is the final value of the input.
  • If the input is linked to an output on an upstream node, then the upstream node's value is merged into the default value to produce the final value of the input. The merging process is described further in the section on links below.

Cleaning Nodes

Cleaning a node forces the node to perform its computation. A computation is an operation where it uses its input values to compute its output values.

In order for a node to output a clean (up to date) value, its inputs must be clean as well. Hence when you a clean a node, all of the linked upstream nodes will be cleaned recursively as well.

After a node is cleaned it will be marked on the right with the letter "C". When a node is dirty, the letter "C" will be absent.


Viewing Outputs

After a node is cleaned, you can view the latest output values produced by the node. Nodes can output any of the usual data types: numbers, strings, boolean, objects and arrays. The user interface will allow you to dive into objects and arrays by double clicking on them.

You can also view the outputs of a dirty node. However the values will no longer be current or up to date.

The current node being viewed will be marked on the right with the letter "V".


Links

A link is used to direct output data from an output on an upstream node to an input on a downstream node. If the data through the link matches the default value of an input in terms of its structure and type, the data flowing through the link will get merged into the input's default value to produce the final value. Otherwise it will get ignored.

These are the rules that determine how the data from an upstream node merges into the default value of an input to produce the final value.

  • If both types are either numbers, strings or booleans then the upstream value will override the downstream default value.

  • If both types are objects, the properties of the upstream object will override the properties of the same name on the downstream default value, if the property values have the same matching internal structures/types. If the property does not exist on the downstream default value, it will get added.

  • If both types are arrays, the first element of the default value will determine the structure and type of the elements that it will accept as overrides. If the default value is an empty array, then any array will be accepted and become the final value.

  • In all other cases, the upstream value has no effect. The final value will be the default value without modification.


Group Nodes

A group node is a node which contains another node graph inside itself. When you dive into a group node, you will see the contents of the group node. When you surface from a group you will see the node graph in which the group node exists.

Group nodes allow you to collapse a complex node graph into one node and to also define a strict interface to the logic contained within. The interface is represented by the inputs and outputs on the group node.

In order to create inputs or outputs on a group node, you will first need to dive into the group. Then create Input Nodes or Output Nodes respectively. Each Input Node or Output Node inside the group will appear as inputs or outputs respectively on the group node under the same names.

There are a few different types of group nodes. Each provides a specialized context for their nodes to operate in. They are data group nodes, script group nodes and web group nodes.


Data Group Nodes

The sole purpose of the Data Group Node is to encapsulate a node graph which manipulates data. As such, if you are inside a Data Group, you can only create nodes related to data creation, data modification, and control flow logic. In particular you will not be able to create nodes that belong inside the Web Group, because they require the use of the browser which only the Web Group provides.

Data manipulation nodes have no side effects. They are purely functional in that they take inputs into consideration and then produce outputs. As long as the inputs are the same they will always produce the same outputs.

To create nodes from inside a Data Group, simply right-click on empty space in the node graph. This will bring up a menu containing items to create nodes.


Script Group Nodes

The sole purpose of the Script Group Node is to allow the user to specify custom logic in the node graph. This custom logic functions just like a javascript expression or a script.

Users can specify their custom logic by setting the default value on the "script" input of the Script Group Node. This input is evaluated as javascript to compute the outputs. The input values are added to the script evaluation context under variables named after their input names.

Note that any scripts you specify should be free of side effects. That is, given the same inputs it should always produce the same outputs. Smash Browse assumes that all nodes besides the web manipulation nodes are free of side effects.

If you are inside a Script Group, you can only create Input and Output Nodes. This defines the interface to the script, which essentially defines the input values made available to the script. Other nodes are not permitted because the Script Group Node's sole purpose is to evaluate its script.

To create nodes from inside a Script Group, simply right-click on empty space in the node graph. This will bring up a menu containing items to either create an Input Node or an Output Node.


Web Group Nodes

The sole purpose of the Web Group Node is to encapsulate nodes which manipulate the browser. Browser manipulation nodes manipulate the browser as a side effect. Unlike data manipulation nodes they are not purely functional. Given the same inputs, a browser manipulation node can produce different outputs. This is the main reason why other nodes are excluded from existing inside of Web Group Nodes. It allows us to encapsulate this functionally inpure piece of logic, into something that is purely functional.

When diving into a Web Group Node, a browser window will open automatically. Internal nodes will manipulate this browser as a side effect. When surfacing from inside a Web Group Node, the browser window will close automatically. This opening and closing of the browser is what resets the mutable state (side effects), so that the Web Group Node as a whole becomes purely functional, even though its internal nodes are not.

To summarize, when you are inside a Web Group, you can only create nodes related to web browser manipulation. In particular you won't be able to use control flow logic nodes. However you can simply output the data out of the Web Group Node through its outputs, and then operate on it inside a regular Data Group.

Unlike other group nodes, to create nodes from inside a Web Group, you will need to right-click inside the browser. This will bring up a menu containing items to create web manipulation nodes. The position of the right-click in the browser is important. It will be used to automatically set a number of input values on the nodes it creates. See the section on Automating the Browser for more details.


Posts

A Post Node allows you to extract some data that is flowing through a link, and to post that data to a page inside of Smash Browse. This is similar to posting a status update on social media, except it's for yourself and only you can see it.

This allows you to verify that your automation completed successfully or to notify you when certain events of importance occur inside your node graph.

For example you can post bank account balances, recent price changes on items you're interested in, recent news posts or social media updates, etc.


JavaScript

There are a couple of places where javascript can be used.

The first is when specifying a default value for an input. Any valid expression will work. A few examples follow.


        123 + 87
                    

        {"alpha": 123, "gamma": "hello", "beta": true} 
                    

        [1, true, "hello", {"alpha": 123, "gamma": "hello"}, [2, 3] ]

The second is when specifying the script for a Script Group Node. The Script Group Node allows you to use javascript to compute the output values on the node. All of the input values on the group are made available to the javascript environment. A few examples follow for a node with two inputs (named "input" and "input1") and one output value (named "output").


        var sum = input + input1;
        set_output("output", sum);
                    

        var data = {};
        data.alpha = input;
        data.beta = input1;
        set_output("output", data);

Automating the Browser

These are the basic steps to automating the browser. Watching these video demos will likely make things much clearer.

  • To control the browser you'll first need to create a Web Group Node. Then you'll need to dive into it, as the web manipulation nodes can only be created inside Web Group Nodes.

  • Once you dive into the Web Group you'll see a browser window open up automatically. There may be a popup in the browser telling you to wait, while it loads the first web page.

  • We are now ready to create nodes. Unlike the Data Group where you right-click in the node graph to bring up a menu to create nodes, when you're inside a Web Group you right-click in the browser.

  • The position of the right-click is significant as it allows Smash Browse to automatically determine and set the input values on the nodes that it creates for you. This makes the creation of nodes much quicker. Otherwise you'd need to manually edit each node to set appropriate values for the behavior you want, by yourself.

  • Smash Browse uses a three step process to perform an action. You can consider these as three steps of indirection, compared to just manually performing the action in the web page (like clicking a link). However this indirection allows us to encode more of the symantic meaning of our actions, and makes our automations much more robust to web page changes.

    • First, you need to create a Set of Elements in the browser. This set can contain just one element or it can contain many elements. An element in Smash Browse is either an image, a piece of text, an input field, or a select dropdown. Once an element is in a set it become highlighted with a colored rectangular box around it. Menu items which create or modify sets will have the word, Set, in their name.

    • Second, you need to choose a single element or a set of elements to perform the next action on. This is done automatically when you right-click over an element to bring up the menu. A menu item which operates on single elements will use that element directly. While a menu item which operates on sets will operate on the entire set that the element belongs to.

    • Third, you need to choose what type of action you want to perform on the element or set. Typical set operations include actions like shifting the set in a certain direction. For example you may want to shift the set of images you're interested in on the left of the page, to those on the right. Typical element operations include things like clicking it, or extracting the text displayed on it.

  • There are many ways to isolate elements and element sets before performing an action on an element. The most robust way however is to ensure that the element set has only one element in it before you perform an action on the element. This is because the ordering of elements may change in the set as the web page gets updated or rearranged at a later time. And using exta constraints to whittle down the set helps to make identifying the elements more robust.

  • Usually the main goal of the logic in the Web Group, is to extract text or images and to output that through the outputs. This allows you to affect logic in other groups, like Data Groups. For example if you extract the balance in your bank account, you can compare it to another value and notify yourself through a post if it falls below that other value.
